Warp Control CLI Verification Notes

Environment Setup:
- App binary: /workspace/warp/target/debug/warp
- CLI binary: /workspace/warp/target/debug/warpctrl
- Had to install libxkbcommon-x11-0 (libxkbcommon-x11.so.0) for Warp to launch
- Launched with: cd /workspace/warp && PATH=/root/.cargo/bin:$PATH XDG_CONFIG_HOME=/tmp/warpctrl-verify-config XDG_DATA_HOME=/tmp/warpctrl-verify-data XDG_CACHE_HOME=/tmp/warpctrl-verify-cache DISPLAY=:99 target/debug/warp --api-key "$STAGING_USER_WARP_API_KEY"
- Note: XDG_STATE_HOME was NOT overridden, so both app and warpctrl use /root/.local/state/
- warpctrl commands were run without custom XDG vars since the socket is in the default state dir
- External xterm was used for warpctrl invocations via script files to show command+output

Known issues/observations:
- The warpctrl "input insert" initially returned "missing_target" error when run from xterm with custom XDG vars. Fixed by running without XDG overrides (socket in default state dir).
- An accidental agent invocation occurred when typing into xterm while Warp had focus. Resolved by using script-based xterm invocations.
- No shell command output appeared in Warp as a result of staging commands (commands were correctly staged, not executed).
